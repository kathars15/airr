# 雷达-光电标定操作手册

本文档用于现场完成雷达、光电联动跟踪与标定。内容以当前项目 `Src/main2.py`、`Src/core/calibration.py`、`Src/core/interactive_console.py` 的实际逻辑为准。

## 1. 标定目标

本系统的标定分为两类：

- 角度标定：计算雷达相对光电的方位角偏移和俯仰角偏移。
- 位置标定：在光电有有效测距数据时，估计雷达和光电之间的三维相对位置偏差 `dx, dy, dz`。

角度标定结果保存到：

```text
Src/calibration_data/calibration_params.json
```

位置标定结果保存到：

```text
Src/calibration_data/position_offset.json
```

注意：位置标定不是简单的“距离加减一个常数”，而是估计雷达和光电两个传感器坐标系之间的空间偏移。程序后续会用这个偏移把雷达的方位、俯仰、距离一起修正到光电视角。

## 2. 网络配置说明

配置文件位置：

```text
Src/core/app_config.py
```

常用实际设备配置示例：

```python
RADAR_IP = "192.168.0.99"
HOST_IP = "0.0.0.0"
RADAR_PORT = 8080
HOST_PORT = 9000

OPTICAL_IP = "192.168.0.98"
OPTICAL_LOCAL_IP = "192.168.0.9"
OPTICAL_PORT = 9966
```

字段含义：

- `RADAR_IP`：真实雷达设备 IP，主要用于程序主动给雷达发控制包。
- `RADAR_PORT`：真实雷达设备控制端口。
- `HOST_IP`：本程序监听雷达数据的本机 IP。
- `HOST_PORT`：本程序监听雷达数据的本机端口。
- `OPTICAL_IP`：光电设备 IP。
- `OPTICAL_LOCAL_IP`：本机和光电通信使用的本机 IP。
- `OPTICAL_PORT`：光电通信端口。

如果雷达数据由雷达软件转发给本程序，则本程序主要依赖 `HOST_IP` 和 `HOST_PORT`。`RADAR_IP` 不参与接收雷达软件转发数据。

推荐将 `HOST_IP` 设置为：

```python
HOST_IP = "0.0.0.0"
```

这表示监听本机所有 IPv4 地址。无论雷达软件转发到 `127.0.0.1:9000` 还是 `192.168.0.9:9000`，程序都有机会收到。

## 3. 雷达软件转发设置

雷达软件中通常有三块设置：

- 本地设置：雷达软件自己接收真实雷达数据的地址，例如 `192.168.0.9:6000`。
- 雷达设置：真实雷达设备地址，例如 `192.168.0.99:8080`。
- 转发设置：雷达软件把数据转发给本程序的地址，例如 `192.168.0.9:9000` 或 `127.0.0.1:9000`。

本程序只接收“转发设置”发出来的数据。

推荐雷达软件转发设置：

```text
传输协议：UDP
IP地址：192.168.0.9
端口：9000
应用协议：Raw
```

如果转发 IP 使用 `127.0.0.1`，理论上也可以，但要确认雷达软件确实支持回环地址转发。现场更稳的做法是转发到真实网卡 IP `192.168.0.9`，程序监听 `0.0.0.0:9000`。

## 4. 启动前检查

1. 确认本机 IP。

```powershell
ipconfig
```

应能看到接雷达/光电的网卡上有：

```text
192.168.0.9
```

2. 确认雷达软件能显示目标。

雷达控制台里能看到目标，只说明“真实雷达到雷达软件”已通，不代表已经转发给本程序。

3. 用探针确认转发端口有数据。

```powershell
cd D:\desk\airr\airport\XJTU_Fusion_V6.4_box\MHT_Bias\Src
python .\tools\radar_udp_probe.py --host 0.0.0.0 --port 9000 --seconds 20
```

结果判断：

- `packets=0`：本程序没有收到任何转发数据，先检查雷达软件转发是否启用。
- `POINT > 0`：收到点迹，可能生成 `radar_calibration_data.csv`。
- `TRACK > 0`：收到雷达原生航迹，可能生成 `raw_tracks.csv`。
- `UNKNOWN > 0`：收到数据但协议头不在当前支持列表中。

## 5. 数据文件说明

数据目录：

```text
Src/data
```

主要文件：

- `radar_calibration_data.csv`：原始 POINT 点迹记录。
- `raw_tracks.csv`：原始 TRACK 航迹记录。只有雷达输出航迹包时才会生成。
- `track_log.txt`：MHT 融合后的目标列表，交互控制台的 `l/list` 主要依赖它。
- `track_results.json`：MHT 融合结果 JSON。

重要区别：

```text
POINT 点迹 -> radar_calibration_data.csv
TRACK 航迹 -> raw_tracks.csv
MHT 输出  -> track_log.txt / track_results.json
```

如果只有点迹没有航迹，`raw_tracks.csv` 不会生成，这是正常现象。

## 7. 常用命令说明

### 查看目标

```text
l
```

或：

```text
list
```

作用：显示当前 MHT 输出的雷达目标，例如 `Radar-1`、`Radar-3`。

### 手动指定光电跟踪

```text
t 1
```

等价于：

```text
t Radar-1
```

作用：锁定指定目标，并把该目标的方位、俯仰、距离发送给光电。

### 查看光电状态

```text
status
```

作用：查看光电当前方位、俯仰、工作状态和目标数量。

光电状态含义：

```text
0 = 空闲
1 = 搜索
2 = 跟踪
```

标定采样要求光电状态为 `2`。

### 查看雷达和光电对比

```text
p
```

作用：显示当前锁定目标的雷达数据和光电数据对比，便于观察方位、俯仰、距离差异。

### 开始标定

```text
cal 1
```

等价于：

```text
cal Radar-1
```

作用：锁定目标并进入标定模式。`cal` 本身已经包含锁定目标动作，因此不一定要先执行 `t`。

### 查看标定状态

```text
cstat
```

作用：查看是否正在标定、当前目标、雷达样本数等。样本数持续增长，说明采样正在进行。

### 停止标定并计算

```text
done
```

作用：停止采样并计算标定结果。`cal` 不会自动结束，必须手动输入 `done`。

### 查看标定结果

```text
cres
```

作用：显示角度标定结果和位置标定结果。

如果只有角度标定，会看到方位角偏移、俯仰角偏移和样本数。

如果位置标定成功，会看到 `dx, dy, dz`、样本数和 `use_position` 状态。

### 清空标定结果

```text
cclear
```

作用：清除保存的角度标定和位置标定结果，恢复未标定状态。

### 测试标定效果

```text
ctest
```

作用：手动输入雷达方位和俯仰，查看应用当前标定参数后的结果。

### 释放目标

```text
r
```

作用：释放当前目标，清空当前锁定状态，并让光电释放目标。

### 退出程序

```text
q
```

或：

```text
quit
```

## 8. 推荐现场标定流程

1. 启动雷达软件，确认雷达目标正常显示。

2. 确认雷达软件转发到本程序端口。

```powershell
python .\tools\radar_udp_probe.py --host 0.0.0.0 --port 9000 --seconds 20
```

3. 启动主程序。

```powershell
python .\main2.py
```

4. 查看目标列表。

```text
l
```

5. 选择一个稳定、距离适中、持续存在的目标。

示例：

```text
cal 1
```

6. 查看光电状态。

```text
status
```

等待光电状态稳定为：

```text
2 = 跟踪
```

7. 查看样本增长。

```text
cstat
```

建议光电稳定跟踪后继续采样 `10~20` 秒。最低门槛是 5 组样本，但现场建议积累十几组以上。

8. 停止并计算。

```text
done
```

9. 查看结果。

```text
cres
```

10. 观察跟踪效果。必要时使用 `p` 查看雷达和光电差异。

## 9. 什么时候可以 done

满足以下条件再执行 `done`：

- 光电 `status` 持续为 `2`。
- `cstat` 中样本数持续增长。
- 目标 ID 没有频繁变化。
- 目标距离、方位、俯仰变化连续，没有明显跳变。
- 已稳定采样约 `10~20` 秒。

如果过早 `done`，可能出现样本不足或标定结果抖动较大。

## 10. 角度标定和位置标定的区别

角度标定计算：

```text
azimuth_offset
pitch_offset
```

后续修正方式近似为：

```text
修正后方位 = 雷达方位 + azimuth_offset
修正后俯仰 = 雷达俯仰 + pitch_offset
```

位置标定计算：

```text
dx, dy, dz
```

它估计的是光电相对雷达的空间安装偏移。后续会先把雷达方位、俯仰、距离转成三维坐标，再应用位置偏移，最后转回光电视角的方位、俯仰、距离。

位置标定需要光电有有效距离数据。如果光电没有测距，或位置标定失败，程序会退回角度标定。

## 11. 常见问题排查

### 雷达软件有目标，但程序没有文件

先确认是否有数据转发到程序端口：

```powershell
python .\tools\radar_udp_probe.py --host 0.0.0.0 --port 9000 --seconds 20
```

如果 `packets=0`，说明雷达软件没有转发到本程序。

### 有 radar_calibration_data.csv，但没有 raw_tracks.csv

说明收到了 POINT 点迹，但没有收到 TRACK 航迹。`raw_tracks.csv` 只记录原始航迹包，不记录点迹。

### 有 raw_tracks.csv，但 l 看不到目标

可能 MHT 没有输出稳定航迹，或者目标超过 `MAX_RANGE`。

当前 `MAX_RANGE` 位于：

```text
Src/core/app_config.py
```

例如：

```python
MAX_RANGE = 2500
```

`track_log.txt` 只写入距离小于等于 `MAX_RANGE` 的 MHT 目标。

### 光电状态不是 2

标定不会记录有效光电样本。请先确保光电已经锁定目标，并且 `status` 显示跟踪状态为 `2`。

### done 后提示样本不足

通常原因：

- 标定时间太短。
- 光电没有进入跟踪状态。
- 雷达目标丢失或 ID 变化。
- 雷达和光电时间配对失败。

当前雷达/光电配对时间窗口：

```python
CALIBRATION_PAIR_TIME_WINDOW = 0.15
```

即两边测量时间差需要小于约 `0.15s` 才能组成一组样本。

### 转发到 127.0.0.1 收不到

部分雷达软件可能不稳定支持回环地址转发。建议转发到真实网卡 IP：

```text
192.168.0.9:9000
```

程序监听：

```python
HOST_IP = "0.0.0.0"
HOST_PORT = 9000
```

## 13. 现场完整操作示例

本节给出一次完整标定的模拟过程。现场输出可能因为编码显示略有不同，但关键字段含义一致。

### 第一步：确认雷达转发是否进入程序端口

输入：

```powershell
cd D:\desk\airr\airport\XJTU_Fusion_V6.4_box\MHT_Bias\Src
python .\tools\radar_udp_probe.py --host 0.0.0.0 --port 9000 --seconds 20
```

正常示例：

```text
[probe] listening on 0.0.0.0:9000 for 20s
[probe] #1 from 192.168.0.9:6000 bytes=1472 head=0xA0A0A7A7 type=STATUS frame_len=128
[probe] #2 from 192.168.0.9:6000 bytes=1208 head=0xA1A1A8A8 type=POINT frame_len=1208
[probe] done. packets=20, STATUS=10, POINT=10, TRACK=0, END=0, UNKNOWN=0
```

判断：

- `packets > 0`：雷达软件已经转发到程序端口。
- `POINT > 0`：有点迹输入，可能生成 `radar_calibration_data.csv`。
- `TRACK > 0`：有雷达原生航迹输入，可能生成 `raw_tracks.csv`。
- `packets=0`：程序没有收到任何雷达转发数据，不要继续标定，先检查雷达软件转发设置。

异常示例：

```text
[probe] done. packets=0, STATUS=0, POINT=0, TRACK=0, END=0, UNKNOWN=0
```

处理：

```text
检查雷达软件“转发设置”的 IP 和端口。
建议转发到 192.168.0.9:9000，程序监听 0.0.0.0:9000。
确认点击“应用”和“确定”。
```

### 第二步：启动主程序

输入：

```powershell
python .\main2.py
```

正常现象：


判断：

- 出现 `>` 提示符，说明交互控制台已经可用。
- 如果一直没有目标，先看 `Src/data` 目录是否生成 `radar_calibration_data.csv`、`track_log.txt`。

异常示例：

```text
PermissionError: [WinError 5] 拒绝访问
```

处理：

```text
在普通 PowerShell 或管理员 PowerShell 中直接运行 python .\main2.py。
不要在受限沙盒终端里运行完整主程序。
```

### 第三步：查看雷达目标列表

输入：

```text
l
```

正常示例：

```text
Radar-1: 距离=346.2m, 方位=13.0°, 俯仰=12.2°, 速度=4.5m/s
Radar-3: 距离=2263.1m, 方位=226.4°, 俯仰=5.8°, 速度=3.6m/s
```

- 优先选择距离适中、持续存在、速度和角度变化平稳的目标。

异常示例：


判断：

- `cal` 已经包含锁定目标动作，不一定要先执行 `t 1`。
- 如果光电还没有跟上，标定模式会开启，但有效光电样本暂时不会增长。

异常示例：

```text

位置偏移标定参数:
  dx: 1.25m
  dy: -0.48m
  dz: 0.72m
  样本数量: 14
  use_position: True
```

判断：

- 只有角度结果：当前只做了方位/俯仰修正。
- 有 `dx, dy, dz` 且 `use_position=True`：位置标定可用，后续会同时修正方位、俯仰、距离。
- 样本数太少或偏移异常大时，不建议直接使用结果。

### 第十步：测试标定效果

输入：

```text
ctest
```

程序会提示输入雷达方位、俯仰。

示例输入：

```text
雷达方位: 13.0
雷达俯仰: 12.2
```

示例输出：

```text
原始: (13.0°, 12.2°)
标定后: (11.2°, 12.8°)
偏移: 方位=-1.80°, 俯仰=0.60°
```

判断：

- 用来快速确认当前标定参数是否已经生效。
- 如果结果不合理，可以执行 `cclear` 清空后重新标定。

### 第十一步：释放目标或继续下一轮

释放当前目标：

```text
r
```

正常现象：

```text
已释放当前目标
```

重新标定另一个目标：

```text
l
cal 3
cstat
done
cres
```

清空标定结果重新开始：

```text
cclear
```

正常现象：

```text
标定参数已清除
```

## 14. 标定中断场景示例

### 雷达目标丢失

现象：

```text
cstat 显示样本数不再增长
l 中看不到原来的 Radar-1
```

建议操作：

```text
done
cres
```

如果样本不足或结果明显异常：

```text
cclear
l
cal 新目标ID
```

说明：标定目标不会因为自动跟踪切换而自动变成新目标。原来 `cal 1`，即使自动跟踪后来切到 `Radar-3`，标定目标仍然是 `Radar-1`，继续采样通常没有意义。

### 光电没有进入跟踪

现象：

```text
status 显示工作状态为 0 或 1
cstat 样本数不增长或增长很慢
```

建议操作：

```text
t 1
status
```

等状态变为 `2` 后，再重新：

```text
cal 1
```

### 目标 ID 频繁变化

现象：

```text
l 中目标一会儿是 Radar-1，一会儿变成 Radar-8
p 中雷达和光电差值跳变很大
```

建议操作：

```text
r
l
cal 更稳定的目标
```

### 只有点迹没有航迹

现象：

```text
radar_calibration_data.csv 有内容
raw_tracks.csv 没有内容
```

说明：

```text
程序收到了 POINT 点迹，但没有收到 TRACK 航迹。
raw_tracks.csv 只记录原始航迹，不记录点迹。
```

此时仍可能通过 MHT 输出 `track_log.txt`，但如果 `l` 没有目标，就需要等待 MHT 形成稳定航迹，或检查雷达/软件是否输出航迹信息。
